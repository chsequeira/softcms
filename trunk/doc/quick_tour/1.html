Symfony2 - краткий обзор. Часть 1.

January 10th, 2011 (02:57 pm)

Общая картина

Итак, вы хотели бы испробовать в деле Symfony2, но имеете в распоряжении лишь 10 минут? Значит, первая часть этого руководства была написана для вас. В ней объясняется, с чего начать работу с Symfony2 на примере структуры простого готового проекта.
Если вы работали с веб фреймворками и раньше, с Symfony2 вы почувствуете себя в своей тарелке.

1.1. Скачиваем и устанавливаем Symfony2
Для начала, проверьте, что у вас установлен и грамотно сконфигурирован для работы с веб-сервером (к примеру, Apache) как минимум PHP 5.3.2.
Готово? Тогда скачиваем Symfony2. Для более быстрого старта мы будем использовать Symfony2 sandbox (т.н. «песочницу»). Это проект Symfony2, где все необходимые библиотеки и некоторые простые контроллеры уже добавлены. Главное преимущество «песочницы» над другими типами установочных версий — в «песочнице» вы можете сразу же начать работу.
Скачайте sandbox и распакуйте в корневую веб-директорию. Папки должны выглядеть так:

  1. www/ <- ваша корневая веб-директория
  2.  sandbox/ <- нераспакованный архив
  3.         app/
  4.             cache/
  5.             config/
  6.             logs/
  7.         src/
  8.             Application/
  9.                 HelloBundle/
  10.                     Controller/
  11.                     Resources/
  12.             vendor/
  13.                 symfony/
  14.         web/


1.2 Проверяем конфигурацию
Для предотвращения неудобств в дальнейшей работе, убедитесь, что с вашими настройками, проект Symfony2 можно запустить по ссылке:
http://localhost/sandbox/web/check.php

Внимательно прочитайте данные, полученные от скрипта и исправьте все проблемы, которые были найдены.
Теперь запросите вашу первую настоящую страницу, написанную на Symfony2:
http://localhost/sandbox/web/app_dev.php/

Symfony2 следует поздравить вас с быстрой работой!



1.3. Создание вашего первого приложения
В состав «песочницы» входит простое приложение «Hello World», мы его будем использовать для того, чтобы больше узнать о Symfony2. Перейдите по следующей ссылке, чтобы Symfony2 поприветствовала вас (замените Fabien на свое имя):
http://localhost/sandbox/web/app_dev.php/hello/Fabien

Что же здесь происходит? Давайте проанализируем ссылку:
app_dev.php: это фронт-контроллер, единственная точка входа в приложение. Она отвечает на все запросы пользователей.

/hello/Fabien: это «виртуальный» путь к ресурсам, доступ к которым хочет получить пользователь.
Ваша задача, как разработчика, - написать такой код, который бы отображал запрос пользователя (/hello/Fabien) к ресурсу, с ним связанному (Hello Fabien!).

1.3.1. Настройка
Но как работает маршрутизация запросов к коду в Symfony2? Всего лишь читая некоторые файлы настройки.
Все файлы конфигурации Symfony2 могут быть написаны в форматах PHP, XML или YAML (YAML — простой формат, делающий описание настроек конфигурации очень легким).

В «Песочнице» по умолчанию стоит YAML, но вы можете легко переключиться на PHP или XML, редактируя файл app/AppKernel.php. Вы можете попробовать переключить форматы сейчас, загляните в конец файла для инструкций (данное руководство показывает настройки для всех поддерживаемых форматов).

1.3.2. Маршрутизация
Symfony2 отправляет запрос, основываясь на файле настроек маршрутизации:
  1. # app/config/routing.yml
  2. homepage:
  3.     pattern:  /
  4.     defaults: { _controller: FrameworkBundle:Default:index }
  5.  
  6. hello:
  7.     resource: HelloBundle/Resources/config/routing.yml


Первые несколько строчек файла настройки маршрута определяют, какой код вызывать, когда пользователь запросит ресурс «/». Более интересна вторая часть, которая импортирует другой файл настройки маршрутизации, что выглядит как:

  1. # src/Application/HelloBundle/Resources/config/routing.yml
  2. hello:
  3.     pattern:  /hello/:name
  4.     defaults: { _controller: HelloBundle:Hello:index }


Ну, и понеслась! Как видите, паттерн ресурса “/hello/:name” (строчка, начинающаяся с двоеточия, например, :name, - заполнитель) сопоставляется с контроллером, который ссылается на значение _controller.

1.3.3. Контроллеры
Контроллер занимается возвращением представления ресурса (большей частью в виде HTML) и определяется как PHP класс:

  1. // src/Application/HelloBundle/Controller/HelloController.php
  2.  
  3.  namespace Application\HelloBundle\Controller;
  4.  
  5.  use Symfony\Bundle\FrameworkBundle\Controller\Controller;
  6.  
  7.  class HelloController extends Controller
  8.  {
  9.      public function indexAction($name)
  10.      {
  11.          return $this->render('HelloBundle:Hello:index.twig', array('name' => $name));
  12.  
  13.          // render a PHP template instead
  14.          // return $this->render('HelloBundle:Hello:index.php', array('name' => $name));
  15.      }
  16.  }


Код достаточно простой, но давайте объясним каждую его строчку:
Строка 3: Symfony2 использует преимущества фичей PHP 5.3 и, как следствие, все контроллеры именованы соответственно (пространство имен — первая часть значения маршрута контроллера: Hello Bundle).
Строка 7: Название контроллера — конкатенация второй части значения маршрута _controller (Hello) и Controller. Это расширяет встроенный Controller класс, что обеспечивает полезные ссылки (с этим мы столкнемся позже в данном руководстве).
Строка 9: Каждый контроллер состоит из нескольких действий. В соответствии с конфигурацией, страница приветствия обрабатывается действием index (третья часть значения маршрута _controller ). Этот метод получает значения ресурса заполнителя в качестве аргументов ($name — в нашем случае).
Строка 11: Метод render() загружает и рендерит шаблоны (HelloBundle:Hello:index.twig) с переменной, передающейся в качестве второго аргумента.

Но что же такое связка? Весь код, что вы пишете в Symfony2, организован в связки. Говоря о Symfony2, связка — это организованный набор файлов (PHP файлов, стилей, изображений, Javascript, …), который принадлежит одной функции (блог, форум) и которым можно легко поделиться с другими разработчиками. В нашем примере мы имеем только одну связку — HelloBundle.

1.3.4. Шаблоны
Итак, контроллер рендерит шаблон HelloBundle:Hello:index.twig. Но что же в самом имени шаблона? HelloBundle — имя связки, Hello — контроллер, а index.twig — имя шаблона. По умолчанию sandbox использует Twig как средство шаблона:

  1. {# src/Application/HelloBundle/Resources/views/Hello/index.twig #}
  2. {% extends "HelloBundle::layout.twig" %}
  3.  
  4. {% block content %}
  5.     Hello {{ name }}!
  6. {% endblock %}


Поздравляем! Вы посмотрели на свой первый кусок кода, написанный в Symfony2. Это было не так уж трудно, не так ли? Symfony2 делает реализацию быстрых сайтов действительно легкой.

1.4. Работая с окружением
Теперь, когда вы имеете более детальное представление о Symfony2, посмотрите ближе на низ страницы: вы заметите маленькую плашку с логотипами Symfony2 и PHP. Она называется «Тулбар веб отладки» и является лучшим другом разработчика. Конечно, такой инструмент не должен отображаться, когда вы разворачиваете свое приложение на производственном сервере. Вот почему в web/directory (app.php) вы сможете найти другой фронт-контроллер, оптимизированный под окружение сервера:
http://localhost/sandbox/web/app.php/hello/Fabien

Последнее, но не по важности: на производственных серверах вам следует разворачивать корневую директорию в папку web/ для большей безопасности и лучшего вида URL'а:
http://localhost/hello/Fabien

Чтобы сделать производственное окружение как можно более быстрым, Symfony2 хранит кэш в папке the app/cache/. После внесения изменений в конфигурацию кода вам приходится вручную удалять кэш-файлы. Вот почему всегда следует использовать фронт-контроллер для разработки (app_dev.php), когда вы работаете над проектом.

1.5. Заключительные слова
Вот и прошли десять минут. Теперь вы должны уметь создавать собственные простые маршруты, контроллеры и шаблоны. Чтобы поупражняться, попробуйте создать что-нибудь более полезное, чем приложение Hello.
