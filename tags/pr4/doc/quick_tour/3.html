3. Контроллер
Все еще с нами даже после двух частей? Вы становитесь Symfony2-зависимым! Давайте узнаем без промедления, что контроллеры могут сделать для вас.

3.1. Используемые форматы
В наши дни веб-приложения должны уметь доставлять больше, чем просто HTML-страницы. От XML для RSS каналов или веб-сервисов до JSON для Ajax-запросов, и еще множество форматов, есть из чего выбирать. Поддержка этих форматов весьма проста в Symfony2. Отредактируйте routing.yml и добавьте _format со значением для xml:



Вот и всё, что нужно. Нет нужды менять контроллер. Для стандартных форматов Symfony2 будет также выбирать лучшую Content-Type шапку для отклика. Если вы хотите иметь поддержку различных форматов для одного действия, используйте заменяемое значение _format в шаблоне:



Теперь контроллер будет вызываться для URL'ов в виде /hello/Fabien.xml или /hello/Fabien.json. Так как значение по умолчанию для _format — html, /hello/Fabien и /hello/Fabien.html оба подойдут для html формата.



Запись requirements определяет регулярные выражения, которые должны совпасть с заменяемым значением. В этом примере, если вы пытаетесь вызвать ресурс /hello/Fabien.js, вы получите ошибку 404, так как он не совпадает с требованиями _format.

3.6. Сессия
Несмотря на то, что HTTP протокол не имеет статического состояния, Symfony2 обеспечивает хороший объект сессии, который представляет клиента (будь то реальный человек, использующий браузер, или бот, или веб-сервер). Между двумя запросами Symfony2 сохраняет атрибуты в куки, используя нативные PHP сессии.
