2. Вид
После прочтения первой части руководства, вы решили, что Symfony2 оказалась достойна еще десяти минут. В этой второй части вы больше узнаете о шаблонизаторе Twig. Twig — гибкий, быстрый и безопасный шаблонизатор для PHP. Он сделает ваши шаблоны краткими и легко читаемыми, а так же более дружелюбными к веб-дизайнерам.

Вместо Twig вы, конечно, можете использовать для шаблонов просто PHP. Оба шаблонных средства поддерживаются Symfony2 и имеют равные приоритеты.

2.1. Twig, быстрый обзор
Если вы хотите знать Twig, мы настоятельно рекомендуем почитать официальную документацию. Этот раздел — лишь быстрый обзор основных принципов, с изучения которых можно начать работу.

Шаблонизатор Twig — простой текстовый файл, который может сгенерировать любой текстовый формат (HTML, XML, CSV, LaTeX, …). Twig определяет два вида разделителей:

{{ ... }}: вывод переменных или результата выражения в шаблоне
{% ... %}: тэг, контролирующий логику шаблона. Используется для выполнения утверждений вроде петли.
Вот маленький шаблон, демонстрирующий основные принципы:

  1. <!DOCTYPE html>
  2. <html>
  3.     <head>
  4.         <title>My Webpage</title>
  5.     </head>
  6.     <body>
  7.         <h1>{{ page_title }}</h1>
  8.  
  9.         <ul id="navigation">
  10.             {% for item in navigation %}
  11.                 <li><a href="{{ item.href }}">{{ item.caption }}</a></li>
  12.             {% endfor %}
  13.         </ul>
  14.     </body>
  15. </html>


Переменные, передающиеся в шаблон, могут быть строками, массивами или даже объектами. Twig абстрагирует разницу между ними и позволяет получить доступ к свойствам переменной с точечной нотацией (.):

  1. {# array('name' => 'Fabien') #}
  2. {{ name }}
  3.  
  4. {# array('user' => array('name' => 'Fabien')) #}
  5. {{ user.name }}
  6.  
  7. {# force array lookup #}
  8. {{ user['name'] }}
  9.  
  10. {# array('user' => new User('Fabien')) #}
  11. {{ user.name }}
  12. {{ user.getName }}
  13.  
  14. {# force method name lookup #}
  15. {{ user.name() }}
  16. {{ user.getName() }}
  17.  
  18. {# pass arguments to a method #}
  19. {{ user.date('Y-m-d') }}


Важно знать, что фигурные скобки не являются частью переменной, но лишь оператором печати. Если вы вызываете переменную внутри тэгов, не ставьте фигурные скобки.



2.2. Декорирование шаблонов
Чаще всего, шаблоны в проектах имеют общие элементы, как всем известные «шапка» и «подвал». В Symfony2 мы подошли к этой проблеме по-другому: шаблон может быть декорирован другим шаблоном. Это работает так же, как в PHP классах: наследование шаблонов позволяет вам создать базовый (основной) шаблон-слой, который будет содержать все общие элементы вашего сайта и установить блоки, которые дочерним шаблонам позволено переопределять.

Шаблон index.twig наследует от layout.twig благодаря тэгу extends:

  1. {# src/Application/HelloBundle/Resources/views/Hello/index.twig #}
  2. {% extends "HelloBundle::layout.twig" %}
  3.  
  4. {% block content %}
  5.     Hello {{ name }}!
  6. {% endblock %}


Нотация HelloBundle::layout.twig — звучит знакомо, не так ли? Это такая же нотация, как при привязке к шаблону. Часть :: означает, что элемент контроллера пуст, так что соответствующий файл хранится непосредственно в views/.
Теперь давайте посмотрим на файл layout.twig:

  1. {% extends "::layout.twig" %}
  2.  
  3. {% block body %}
  4.     <h1>Hello Application</h1>
  5.  
  6.     {% block content %}{% endblock %}
  7. {% endblock %}


Тэги {%block%} определяют два блока (body и content), которые дочерний шаблон может заполнить. Все, что делает тэг блока — говорит шаблонизатору, что дочерний шаблон может переопределять эти части шаблона. Шаблон index.twig переопределяет блок content. А другой определен в базовой слое как сам слой, декорированная еще одним шаблоном.

Twig поддерживает множественные уровни декорирования: слой сам по себе может быть декорирован другой подложкой. Когда часть-связка имени шаблона пуста (::layout.twig), вид ищется в папке app/views/. Эта директория хранит в себе глобальные виды для всего проекта:

  1. {# app/views/layout.twig #}
  2. <!DOCTYPE html>
  3. <html>
  4.     <head>
  5.         <meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
  6.         <title>{% block title %}Hello Application{% endblock %}</title>
  7.     </head>
  8.     <body>
  9.         {% block body '' %}
  10.     </body>
  11. </html>


2.3. Специальные тэги и фильтры
Одна из лучших особенностей Twig — его расширяемость через новые тэги и фильтры. Symfony2 выпущена связанной со многими специальными тэгами и фильтрами, облегчающими работу дизайнера.

2.3.1. Подключение других шаблонов
Лучший способ разделить кусок кода между несколькими различными шаблонами — определить шаблон, который может быть включен в другой.
Создаем шаблон hello.twig:

  1. {# src/Application/HelloBundle/Resources/views/Hello/hello.twig #}
  2. Hello {{ name }}


И меняем шаблон index.twig, чтобы включить его туда:

  1. {# src/Application/HelloBundle/Resources/views/Hello/index.twig #}
  2. {% extends "HelloBundle::layout.twig" %}
  3.  
  4. {# override the body block from index.twig #}
  5. {% block body %}
  6.     {% include "HelloBundle:Hello:hello.twig" %}
  7. {% endblock %}


2.3.2. Вложение других контроллеров
А что, если вы захотите вставить результат другого контроллера в шаблон? Это очень полезно при работе с Ajax или когда вставляемый шаблон нуждается в некоторой переменной, недоступной в основном шаблоне.

Если вы создаете действие fancy и хотите включить его в шаблон index.twig, используйте тэг render:

  1. {# src/Application/HelloBundle/Resources/views/Hello/index.twig #}
  2. {% render "HelloBundle:Hello:fancy" with ['name': name, 'color': 'green'] %}


Здесь строка HelloBundle:Hello:fancy относится к действию fancy контроллера Hello, и аргумент используется в качестве имитируемых значений к пути запроса.

  1. // src/Application/HelloBundle/Controller/HelloController.php
  2.  
  3. class HelloController extends Controller
  4. {
  5.     public function fancyAction($name, $color)
  6.     {
  7.         // create some object, based on the $color variable
  8.         $object = ...;
  9.  
  10.         return $this->render('HelloBundle:Hello:fancy.twig', array('name' => $name, 'object' => $object));
  11.     }
  12.  
  13.     // ...
  14. }


2.3.3. Создаем ссылки между страницами
Говоря о веб-приложениях, создание ссылок между страницами является необходимостью. Вместо кодирования URL'ов в шаблонах, тэг path знает, как сгенерировать URL, основанный на настройках маршрута. Таким образом, все ваши ссылки могут быть легко обновлены путем изменения настроек:

  1. <a href="{% path 'hello' with ['name': 'Thomas'] %}">Greet Thomas!</a>


Тэг path использует имя маршрута и массив параметров в качестве аргументов. Название маршрута — основной ключ, в соответствии с которым ссылаются маршруты, а параметры — значения заменяемых символов, определенные в маршруте паттерна:

  1. # src/Application/HelloBundle/Resources/config/routing.yml
  2. hello: # The route name
  3.     pattern:  /hello/:name
  4.     defaults: { _controller: HelloBundle:Hello:index }


Вы также можете создать абсолютные ссылки с тэгом url:

  1. {%url'hello'with['name':'Thomas']%}.


2.3.4. Использование других средств: изображения, JavaScript, стили
Чем бы был Интернет без картинок, JavaScript'а и стилей? В Symfony2 есть три помощника, с которыми будет легче работать: assets, javascripts и stylesheets:

  1. <link href="{% asset 'css/blog.css' %}" rel="stylesheet" type="text/css" />
  2.  
  3. <img src="{% asset 'images/logo.png' %}" />


Основная задача тэга asset — сделать ваше приложение более мобильным. Благодаря этому тэгу вы можете перемещать корневую директорию приложения куда угодно в вашей корневой веб-директории без изменений в коде шаблона.

2.4. Экранирование
Twig экранирует все по умолчанию. Читайте документацию для того, чтобы узнать больше об экранировании и расширениях Escaper.

2.5. Заключительные слова
Twig —очень простой, но мощный в работе инструмент. Благодаря встроенным подложкам, блокам, шаблонам и действиям стало легко сделать ваши шаблоны легкими и расширяемыми.
Вы проработали с Symfony2 всего двадцать минут, и уже можете делать с его помощью интересные и забавные штуки. В этом вся сила Symfony2. Приходя к выводу, что основные элементы легки для изучения, скоро вы узнаете, что эта легкость спрятана под очень гибкой архитектурой.
